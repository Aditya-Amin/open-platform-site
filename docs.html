The Content API

The Guardian's content API is the tool we use to drive our website and apps, and you can make use of it too.

If you don't already have one, you'll need to get an access key for the API. After that you can begin to write your own queries and use our data in your own applications.

We make use of user tiers that are bound to access keys to control which content can be retrieved from the API, and the tier you require will depend on the content you intend to serve through your application.

User Tiers

There are two tiers for public consumption: Developer and Rights-managed.

Developer

As a developer you'll have access to content which is not reserved for the rights-managed tier. At this level you'll get all the fields and metadata you'll need to construct most types of applications.

However, you will find that some or all of the media elements that you'll see on our website will not be provided in the API's responses to developer tier requests. If you require access to this data please contact our commercial department to discuss ways in which we may be able to help you.

Rights-managed

As a rights-managed user, you will have access to everything in the developer tier plus the rich media elements that we display on our site and in our mobile apps.

If your application needs to make use of this additional content, you will need to obtain a rights-managed access key. Please contact our commercial department to discuss the nature of your intended application and the specifics of the rights-management controls.

We provide four endpoints through which you can retrieve our content;

  • /search
  • /tags
  • /sections
  • / (root)

You can take some control over the amount and type of data that is returned in the responses to your queries by using a show-fields parameter in many scenarios.

If your application needs to regularly poll the API for updated content, there are a few things you should know. Please ensure that you read the guide for polling applications before starting.

show-fields

Our content contains many different fields, and by default none of these are included in API responses to your requests.

You can ask for only the specific fields that you're interested in, and that's what we'd recommend you do to reduce the amount of parsing you'll have to perform otherwise.

However, if you just want everything that's available to your user tier you can simply request all fields.

As an example, if you visit

content.guardianapis.com/technology/2014/may/20/samsung-google-glass-competitor-gear-blink-trademark?api-key=my-api-key&show-fields=body

you'll see that the output contains a fields collection with a single member: body. There are several fields available to you, depending on your user-tier. See the advanced show-fields help for details.

The / (root) endpoint, sometimes also referred to as the item endpoint, is used to retrieve a specific piece of content, using its unique id.

Construction

content.guardianapis.com/id?api-key=my-api-key[&show-fields=<list of fields>|fieldName|all][&show-tags=<list of tags>|tagName|all][&show-elements=<list of element types>|element type|all][&show-references=<list of references>|specific reference|all][&show-story-package=true|false][&show-editors-picks=true|false][&show-most-viewed=true|false][&show-related-content=true|false][&hide-recent-content=true|false]

Example

http://content.guardianapis.com/business/2014/feb/18/uk-inflation-falls-below-bank-england-target?api-key=my-api-key

This request will retrieve the single content item identified by its id (business/2014/...-target), shown in bold. In this example that would be an article, but you can also retrieve other content types such as a specific tag or section.

What's a tag?

A tag is a piece of data that we use to categorise our content. We use many different tags so understanding what they mean is important, and with new ones being added all the time you'll want to make sure to keep up to date with the changes.

For example, this article: /lifeandstyle/2014/may/14/recycling-saving-energy-reducing-waste-how-is-it-going-for-you contains many tags including;

  • environment/recycling
  • environment/plasticbags
  • environment/energyefficiency

You can use these tags in your own queries to find other content that has the same tags.

What's a section?

We use sections to logically group our content. For example, you'll find that within our Technology section, technologically-oriented content will be clustered such as items covering games, iPhone, Sony, Google and others.

For example: /technology contains technology-related content (link opens in a new tab/window).

Polling Applications

Things you should know

The key that you are assigned for your user tier is rate-limited and as such any applications that depend on making large numbers of requests on a polling basis are likely to exceed their daily quota and thus be prevented from making further requests until the next period begins.

If you require an elevated limit on requests-per-day or requests-per-second this may be possible to arrange. Please contact us to discuss the nature of your application and the requests you are intending to make.
