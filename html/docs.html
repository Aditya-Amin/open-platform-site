The Content API

The Guardian's content API is the tool we use to drive our website and apps, and you can make use of it too.

If you don't already have one, you'll need to get an access key for the API. After that you can begin to write your own queries and use our data in your own applications.

We offer four endpoints that you can interact with to retrieve our content;

  • /search
  • /tags
  • /sections
  • / (root)

We make use of access tiers that control which content can be retrieved from the API. There's more information about that here.

You can control how much data is returned in the responses to your queries. For more information about what's available and how to request it, click here.

The /search endpoint with no parameters will return a paginated list of the most recent items, the default results will contain 10 items. Try it.

What's a tag?

A tag is a piece of data that we use to categorise our content. We use many different tags so understanding what they mean is important, and with new ones being added all the time you'll want to make sure to keep up to date with the changes.

For example, this article: /lifeandstyle/2014/may/14/recycling-saving-energy-reducing-waste-how-is-it-going-for-you contains many tags including;

  • environment/recycling
  • environment/plasticbags
  • environment/energyefficiency

You can use these tags in your own queries to find other content that has the same tags.

What's a section?

We use sections to logically group content. For example, you'll find that within our Technology section, technologically-oriented content will be clustered such as items covering games, iPhone, Sony, Google and others.

For example: /technology contains technology-related content.
